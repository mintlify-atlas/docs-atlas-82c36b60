<Warning>
  Mesh shader support in wgpu is **experimental** and subject to breaking changes. The API may have significant bugs. Feedback should be posted on the [mesh shading tracking issue](https://github.com/gfx-rs/wgpu/issues/7197).
</Warning>

Mesh shaders replace the traditional vertex/geometry shader pipeline with a more flexible, compute-like model. Instead of the CPU issuing draw calls that process vertices one by one, mesh shaders let the GPU decide how to subdivide and render geometry — making them well suited for **meshlet rendering**, per-primitive culling, and dynamic level-of-detail.

## When to use mesh shaders

Mesh shaders are most effective when:

- Rendering scenes with large numbers of small geometry groups (meshlets)
- Performing GPU-side culling of back-facing or occluded primitives
- Implementing dynamic LOD based on distance from the camera
- Replacing vertex + geometry shader combinations

Scenes bottlenecked by fragment processing or post-processing will see little benefit.

## Required features

```rust
let (device, queue) = adapter
    .request_device(&wgpu::DeviceDescriptor {
        required_features: wgpu::Features::EXPERIMENTAL_MESH_SHADER
            | wgpu::Features::PASSTHROUGH_SHADERS, // required on Metal
        required_limits: wgpu::Limits::defaults()
            .using_recommended_minimum_mesh_shader_values(),
        ..Default::default()
    })
    .await
    .unwrap();
```

Additional optional features:

- `Features::EXPERIMENTAL_MESH_SHADER_MULTIVIEW` — mesh shaders with multiview rendering
- `Features::EXPERIMENTAL_MESH_SHADER_POINTS` — mesh shaders outputting point primitives

## Pipeline overview

A mesh shader pipeline replaces the vertex stage:

```
[draw_mesh_tasks] → [Task shader (optional)] → [Mesh shader] → [Fragment shader]
```

1. **Task shader** (`@task`): a compute-like stage. Each workgroup receives a position in the task grid, optionally reads/writes a shared `task_payload`, and returns a `vec3<u32>` specifying how many mesh shader workgroups to dispatch.
2. **Mesh shader** (`@mesh`): receives the task payload (if any) and produces a list of output vertices and primitives, which are passed directly to the rasterizer.
3. **Fragment shader** (`@fragment`): receives interpolated vertex outputs plus optional non-interpolated per-primitive values.

If there is no task shader, the dimensions passed to `draw_mesh_tasks` specify the mesh shader grid directly.

## Creating a mesh pipeline

Use `Device::create_mesh_pipeline` instead of `create_render_pipeline`:

```rust
let pipeline = device.create_mesh_pipeline(&wgpu::MeshPipelineDescriptor {
    label: Some("my mesh pipeline"),
    layout: Some(&pipeline_layout),
    task: Some(wgpu::TaskState {
        module: &shader_module,
        entry_point: Some("ts_main"),
        compilation_options: Default::default(),
    }),
    mesh: wgpu::MeshState {
        module: &shader_module,
        entry_point: Some("ms_main"),
        compilation_options: Default::default(),
    },
    fragment: Some(wgpu::FragmentState {
        module: &shader_module,
        entry_point: Some("fs_main"),
        compilation_options: Default::default(),
        targets: &[Some(surface_format.into())],
    }),
    primitive: wgpu::PrimitiveState {
        cull_mode: Some(wgpu::Face::Back),
        ..Default::default()
    },
    depth_stencil: None,
    multisample: Default::default(),
    multiview: None,
    cache: None,
});
```

## Dispatching mesh tasks

Replace `draw` / `draw_indexed` with `draw_mesh_tasks`:

```rust
// In a render pass:
render_pass.set_pipeline(&mesh_pipeline);
render_pass.draw_mesh_tasks(x, y, z); // number of task (or mesh) workgroups
```

For GPU-driven rendering, use the indirect variants:

```rust
render_pass.draw_mesh_tasks_indirect(&indirect_buffer, offset);
render_pass.multi_draw_mesh_tasks_indirect(&indirect_buffer, offset, count);
render_pass.multi_draw_mesh_tasks_indirect_count(
    &indirect_buffer, offset,
    &count_buffer, count_offset,
    max_count,
);
```

## WGSL mesh shader syntax

Mesh shaders require the `enable wgpu_mesh_shader;` directive at the top of the WGSL source.

### Full example: task + mesh + fragment

```wgsl
enable wgpu_mesh_shader;

const positions = array(
    vec4(0., 1., 0., 1.),
    vec4(-1., -1., 0., 1.),
    vec4(1., -1., 0., 1.),
);
const colors = array(
    vec4(0., 1., 0., 1.),
    vec4(0., 0., 1., 1.),
    vec4(1., 0., 0., 1.),
);

struct TaskPayload {
    colorMask: vec4<f32>,
    visible: bool,
}

struct VertexOutput {
    @builtin(position) position: vec4<f32>,
    @location(0) color: vec4<f32>,
}

struct PrimitiveOutput {
    @builtin(triangle_indices) indices: vec3<u32>,
    @builtin(cull_primitive) cull: bool,
    @per_primitive @location(1) colorMask: vec4<f32>,
}

struct PrimitiveInput {
    @per_primitive @location(1) colorMask: vec4<f32>,
}

var<task_payload> taskPayload: TaskPayload;
var<workgroup> workgroupData: f32;

// Task shader: dispatches mesh workgroups and writes the task payload
@task
@payload(taskPayload)
@workgroup_size(1)
fn ts_main() -> @builtin(mesh_task_size) vec3<u32> {
    workgroupData = 1.0;
    taskPayload.colorMask = vec4(1.0, 1.0, 0.0, 1.0);
    taskPayload.visible = true;
    return vec3(1, 1, 1); // dispatch a 1×1×1 mesh shader grid
}

struct MeshOutput {
    @builtin(vertices)        vertices:        array<VertexOutput, 3>,
    @builtin(primitives)      primitives:      array<PrimitiveOutput, 1>,
    @builtin(vertex_count)    vertex_count:    u32,
    @builtin(primitive_count) primitive_count: u32,
}

var<workgroup> mesh_output: MeshOutput;

// Mesh shader: produces vertices and primitives
@mesh(mesh_output)
@payload(taskPayload)
@workgroup_size(1)
fn ms_main() {
    mesh_output.vertex_count = 3u;
    mesh_output.primitive_count = 1u;

    mesh_output.vertices[0].position = positions[0];
    mesh_output.vertices[0].color = colors[0] * taskPayload.colorMask;
    mesh_output.vertices[1].position = positions[1];
    mesh_output.vertices[1].color = colors[1] * taskPayload.colorMask;
    mesh_output.vertices[2].position = positions[2];
    mesh_output.vertices[2].color = colors[2] * taskPayload.colorMask;

    mesh_output.primitives[0].indices = vec3<u32>(0, 1, 2);
    mesh_output.primitives[0].cull = !taskPayload.visible;
    mesh_output.primitives[0].colorMask = vec4<f32>(1.0, 0.0, 1.0, 1.0);
}

// Fragment shader: receives per-vertex and per-primitive data
@fragment
fn fs_main(vertex: VertexOutput, primitive: PrimitiveInput) -> @location(0) vec4<f32> {
    return vertex.color * primitive.colorMask;
}
```

### Key WGSL attributes

| Attribute | Stage | Description |
|-----------|-------|-------------|
| `@task` | Task | Marks a task shader entry point |
| `@mesh(VAR)` | Mesh | Marks a mesh shader entry point; `VAR` is the workgroup output variable |
| `@payload(VAR)` | Task, Mesh | Links the task payload global variable |
| `@workgroup_size(x, y, z)` | Task, Mesh | Workgroup dimensions |
| `@builtin(mesh_task_size)` | Task return | How many mesh workgroups to dispatch |
| `@builtin(vertex_count)` | Mesh output | Number of active vertices |
| `@builtin(primitive_count)` | Mesh output | Number of active primitives |
| `@builtin(triangle_indices)` | Primitive | `vec3<u32>` vertex indices for a triangle |
| `@builtin(cull_primitive)` | Primitive | `bool`; if true, the primitive is skipped |
| `@per_primitive` | Fragment input | Non-interpolated per-primitive value from mesh shader |

## Limits

| Limit | Description |
|-------|-------------|
| `max_task_mesh_workgroup_total_count` | Maximum product of x·y·z dimensions in `draw_mesh_tasks` |
| `max_task_mesh_workgroups_per_dimension` | Maximum value for each individual dimension |
| `max_task_invocations_per_workgroup` | Maximum total threads in a task workgroup |
| `max_mesh_invocations_per_workgroup` | Maximum total threads in a mesh workgroup |
| `max_task_payload_size` | Maximum size of `var<task_payload>` in bytes |
| `max_mesh_output_vertices` | Maximum vertices a single mesh workgroup may output |
| `max_mesh_output_primitives` | Maximum primitives a single mesh workgroup may output |

## Platform support

<Tabs>
  <Tab title="Vulkan">
    Full native WGSL support via the `VK_EXT_mesh_shader` extension. Best supported on NVIDIA Turing+ and AMD RDNA2+ GPUs.
  </Tab>
  <Tab title="DX12">
    Full native WGSL support with Shader Model 6.5 and DXC. Requires a compatible GPU and the DXC compiler. Also supports passthrough HLSL shaders.
  </Tab>
  <Tab title="Metal">
    Supported via passthrough MSL shaders (requires `Features::PASSTHROUGH_SHADERS`). WGSL-to-MSL translation for mesh shaders is not yet complete in Naga.
  </Tab>
</Tabs>
