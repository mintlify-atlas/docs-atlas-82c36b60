---
title: "Compute shaders"
description: "Run general-purpose GPU computations with compute pipelines"
---

Compute shaders let you run data-parallel programs on the GPU without drawing any pixels. Common uses include image processing, physics simulation, and machine learning inference.

This guide is based on the [`hello_compute`](https://github.com/gfx-rs/wgpu/tree/trunk/examples/standalone/01_hello_compute) example, which takes a list of floats and doubles each one on the GPU.

## When to use compute

Use a compute pipeline when:
- You need flexible read/write access to multiple buffers from a single shader
- You are transforming data without rasterizing geometry
- You want to generate data for later use in a render pass

## Project setup

```toml
[dependencies]
bytemuck = "1"
env_logger = "0.11"
pollster = "0.4"

```

## Steps

<Steps>
  <Step title="Write the WGSL compute shader">
    Create `src/shader.wgsl`:

    ```wgsl
    @group(0) @binding(0)
    var<storage, read> input: array<f32>;

    @group(0) @binding(1)
    var<storage, read_write> output: array<f32>;

    @compute @workgroup_size(64)
    fn doubleMe(@builtin(global_invocation_id) global_id: vec3<u32>) {
        let array_length = arrayLength(&input);
        if (global_id.x >= array_length) {
            return;
        }
        output[global_id.x] = input[global_id.x] * 2.0;
    }
    ```

    `@workgroup_size(64)` means each workgroup runs 64 threads. The bounds check guards against extra invocations when the input length is not a multiple of 64. A workgroup size of 64 maps well to the hardware warp/wavefront size on most GPUs.
  </Step>

  <Step title="Initialize wgpu (headless)">
    Compute workloads do not need a window. Use `new_without_display_handle`:

    ```rust
    use std::num::NonZeroU64;
    use wgpu::util::DeviceExt;

    env_logger::init();

    let arguments: Vec<f32> = std::env::args()
        .skip(1)
        .map(|s| s.parse().expect("not a float"))
        .collect();

    let instance =
        wgpu::Instance::new(wgpu::InstanceDescriptor::new_without_display_handle());

    let adapter =
        pollster::block_on(instance.request_adapter(&wgpu::RequestAdapterOptions::default()))
            .expect("Failed to create adapter");

    // Verify compute shader support — not guaranteed on all downlevel devices.
    let downlevel_capabilities = adapter.get_downlevel_capabilities();
    if !downlevel_capabilities
        .flags
        .contains(wgpu::DownlevelFlags::COMPUTE_SHADERS)
    {
        panic!("Adapter does not support compute shaders");
    }

    let (device, queue) =
        pollster::block_on(adapter.request_device(&wgpu::DeviceDescriptor {
            label: None,
            required_features: wgpu::Features::empty(),
            required_limits: wgpu::Limits::downlevel_defaults(),
            experimental_features: wgpu::ExperimentalFeatures::disabled(),
            memory_hints: wgpu::MemoryHints::MemoryUsage,
            trace: wgpu::Trace::Off,
        }))
        .expect("Failed to create device");
    ```

    <Note>
      `DownlevelFlags::COMPUTE_SHADERS` may not be set on some OpenGL ES implementations. See [Downlevel capabilities](/guides/platform-support#downlevel-capabilities).
    </Note>
  </Step>

  <Step title="Create storage buffers">
    You need three buffers: one for input data, one for output, and one staging buffer for CPU readback.

    ```rust
    let input_data_buffer = device.create_buffer_init(&wgpu::util::BufferInitDescriptor {
        label: None,
        contents: bytemuck::cast_slice(&arguments),
        usage: wgpu::BufferUsages::STORAGE,
    });

    let output_data_buffer = device.create_buffer(&wgpu::BufferDescriptor {
        label: None,
        size: input_data_buffer.size(),
        usage: wgpu::BufferUsages::STORAGE | wgpu::BufferUsages::COPY_SRC,
        mapped_at_creation: false,
    });

    // MAP_READ cannot be combined with STORAGE — use a separate staging buffer.
    let download_buffer = device.create_buffer(&wgpu::BufferDescriptor {
        label: None,
        size: input_data_buffer.size(),
        usage: wgpu::BufferUsages::COPY_DST | wgpu::BufferUsages::MAP_READ,
        mapped_at_creation: false,
    });
    ```

    `MAP_READ` and `STORAGE` cannot be combined. A separate staging buffer with `COPY_DST | MAP_READ` is required for CPU readback.
  </Step>

  <Step title="Create BindGroupLayout and BindGroup">
    The layout declares what types of resources each binding slot expects. The bind group wires actual buffers to those slots.

    ```rust
    let bind_group_layout =
        device.create_bind_group_layout(&wgpu::BindGroupLayoutDescriptor {
            label: None,
            entries: &[
                wgpu::BindGroupLayoutEntry {
                    binding: 0,
                    visibility: wgpu::ShaderStages::COMPUTE,
                    ty: wgpu::BindingType::Buffer {
                        ty: wgpu::BufferBindingType::Storage { read_only: true },
                        min_binding_size: Some(NonZeroU64::new(4).unwrap()),
                        has_dynamic_offset: false,
                    },
                    count: None,
                },
                wgpu::BindGroupLayoutEntry {
                    binding: 1,
                    visibility: wgpu::ShaderStages::COMPUTE,
                    ty: wgpu::BindingType::Buffer {
                        ty: wgpu::BufferBindingType::Storage { read_only: false },
                        min_binding_size: Some(NonZeroU64::new(4).unwrap()),
                        has_dynamic_offset: false,
                    },
                    count: None,
                },
            ],
        });

    let bind_group = device.create_bind_group(&wgpu::BindGroupDescriptor {
        label: None,
        layout: &bind_group_layout,
        entries: &[
            wgpu::BindGroupEntry {
                binding: 0,
                resource: input_data_buffer.as_entire_binding(),
            },
            wgpu::BindGroupEntry {
                binding: 1,
                resource: output_data_buffer.as_entire_binding(),
            },
        ],
    });
    ```

    The `binding` numbers must match `@binding(N)` in the WGSL shader.
  </Step>

  <Step title="Create the ComputePipeline">
    ```rust
    let module = device.create_shader_module(wgpu::include_wgsl!("shader.wgsl"));

    let pipeline_layout = device.create_pipeline_layout(&wgpu::PipelineLayoutDescriptor {
        label: None,
        bind_group_layouts: &[Some(&bind_group_layout)],
        immediate_size: 0,
    });

    let pipeline = device.create_compute_pipeline(&wgpu::ComputePipelineDescriptor {
        label: None,
        layout: Some(&pipeline_layout),
        module: &module,
        entry_point: Some("doubleMe"),
        compilation_options: wgpu::PipelineCompilationOptions::default(),
        cache: None,
    });
    ```
  </Step>

  <Step title="Encode the compute pass and submit">
    ```rust
    let mut encoder =
        device.create_command_encoder(&wgpu::CommandEncoderDescriptor { label: None });

    {
        let mut compute_pass = encoder.begin_compute_pass(&wgpu::ComputePassDescriptor {
            label: None,
            timestamp_writes: None,
        });

        compute_pass.set_pipeline(&pipeline);
        compute_pass.set_bind_group(0, &bind_group, &[]);

        // Ceiling-divide to cover all inputs with 64-thread workgroups.
        let workgroup_count = arguments.len().div_ceil(64);
        compute_pass.dispatch_workgroups(workgroup_count as u32, 1, 1);
    }
    // Drop compute_pass to end the pass and regain encoder access.

    encoder.copy_buffer_to_buffer(
        &output_data_buffer,
        0,
        &download_buffer,
        0,
        output_data_buffer.size(),
    );

    queue.submit([encoder.finish()]);
    ```

    Nothing executes on the GPU until `queue.submit`. The compute pass and the copy are recorded together and submitted atomically.
  </Step>

  <Step title="Read results: map_async, poll, get_mapped_range">
    ```rust
    let buffer_slice = download_buffer.slice(..);
    buffer_slice.map_async(wgpu::MapMode::Read, |_| {});

    // Block until the GPU finishes and the buffer is mapped.
    // On WebGPU, use the callback instead — device.poll is not available.
    device.poll(wgpu::PollType::wait_indefinitely()).unwrap();

    let data = buffer_slice.get_mapped_range();
    let result: &[f32] = bytemuck::cast_slice(&data);
    println!("Result: {result:?}");
    ```

    `bytemuck::cast_slice` reinterprets the raw `&[u8]` mapping as `&[f32]` with zero copies.
  </Step>
</Steps>

## Run it

```bash
RUST_LOG=wgpu=warn cargo run -- 1.0 2.0 3.0 4.0
# Result: [2.0, 4.0, 6.0, 8.0]
```

<Tip>
  For 2-D problems (image processing), use `@workgroup_size(8, 8)` and dispatch a 2-D grid. Profiling on target hardware is the only reliable way to find the optimal workgroup size.
</Tip>
