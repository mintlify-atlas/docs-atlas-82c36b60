wgpu is a cross-platform, safe, pure-Rust graphics API based on the [WebGPU standard](https://gpuweb.github.io/gpuweb/). It runs natively on Vulkan, Metal, D3D12, and OpenGL, and in browsers via WebGPU or WebGL2 on WebAssembly. It serves as the core of the WebGPU integration in Firefox, Servo, and Deno.

## Architecture

wgpu is structured as a layered stack:

```
Your application
     ↓
wgpu          ← safe Rust API (this crate)
     ↓
wgpu-core     ← validation, resource tracking, WebGPU compliance
     ↓
wgpu-hal      ← unsafe hardware abstraction layer
     ↓
Vulkan / Metal / D3D12 / OpenGL / WebGPU
```

**Key components:**
- **`wgpu`** — the safe public API you program against
- **`wgpu-core`** — implements WebGPU validation and resource management
- **`wgpu-hal`** — thin unsafe abstraction over native GPU APIs
- **`naga`** — shader translator: reads WGSL/GLSL/SPIR-V, writes HLSL/MSL/GLSL/SPIR-V

## Core types

| Type | Description |
|------|-------------|
| `Instance` | Entry point — enumerate adapters and create surfaces |
| `Adapter` | A physical GPU or software renderer |
| `Device` | Open logical connection to a GPU — creates all resources |
| `Queue` | Submit command buffers to the GPU |
| `Surface` | The window or canvas to render to |

## MSRV

The `wgpu` crate requires Rust **1.87** or later. Tests and examples require **1.93**.
